Propriedades

Beeno CRM — Central de Ajuda

No contexto de um CRM, propriedades são campos personalizados ou padrão que armazenam informações sobre os diferentes objetos do sistema, que incluem contatos, negócios e empresas.

O que são propriedades? Propriedades são como os "atributos" ou "campos de informações" que descrevem um objeto em seu CRM. Elas armazenam informações estruturadas e podem ser configuradas para atender às necessidades específicas do seu negócio. Algumas propriedades comuns incluem:

Contatos: Nome, E-mail, Telefone, Cargo, Data de criação. Negócios: Valor, Fase do pipeline, Probabilidade de fechamento, Data de fechamento esperada. Empresas: Nome da empresa, Segmento, Número de Funcionários, Endereço.

Configurar uma propriedade vai além de apenas atribuir um nome, as propriedades podem ter diferentes campos de preenchimento e diferente obrigatoriedades dentro dos objetos atrelados.

Por que as propriedades são importantes? As propriedades são essenciais para a coleta, organização e análise dos dados no CRM, permitindo que as equipes trabalhem de forma mais eficiente e personalizada.

Organizar dados: Assegurar que todas as informações relevantes estejam armazenadas de forma padronizada. Personalizar comunicação: Utilizar as propriedades para criar campanhas direcionadas e personalizadas. Automatizar processos: Configurar gatilhos de automação com base nos valores das propriedades. Gerar relatórios: Criar relatórios e dashboards detalhados com base nos dados armazenados nas propriedades.

Quais os tipos de propriedade eu tenho disponível? As propriedades na Beeno podem assumir diferentes formatos para atender às necessidades de cada negócio. Confira os tipos de propriedade disponíveis:

Data: Armazena datas no formato Brasil (DD/MM/AAAA). Data/Hora: Armazena data e hora no formato Brasil. Seletor Único: Permite escolher apenas uma opção em uma lista de seleção. Seletor Múltiplo: Permite selecionar várias opções em uma lista de seleção. Texto de linha única: Campo para texto curto. Texto de várias linhas: Campo para textos mais longos. Hora: Armazena apenas valores de hora. Número: Campo para valores numéricos. Usuário: Exibe como opções os usuários cadastrados no CRM.

Além das configurações de preenchimento é possível definir se essa propriedade será:

Propriedade Obrigatória Define esta propriedade como obrigatória para o tipo de objeto atrelado a ela. Sendo assim, para a criação deste objeto, obrigatoriamente devemos ter este campo preenchido.

Propriedade Identificador Único Identificador Único é uma chave de valor único para o objeto atrelado a ela, retirando assim a possibilidade de duplicidade de objetos com o mesmo valor numa propriedade de identificador único.

Configurar inapropriadamente "É Identificador Único" para SIM pode causar fusão de contatos em massa e rastreio de atividade do contato incorreto. Configure para SIM apenas se esse campo contém um valor único específico para o contato, tais como e-mail, nome de usuário, etc.

Propriedades à Exibir no Formulário Isso garante que ao criar um destes objetos (contatos, empresas ou negócios) a propriedade será exibida no pop-up/formulário de criação.

Dicas para otimizar o uso de propriedades: Use nomenclaturas claras e padronizadas para facilitar a compreensão. Mantenha as propriedades atualizadas e exclua campos não utilizados. Treine a equipe para preencher as informações de forma consistente. Utilize automações para preencher ou atualizar propriedades automaticamente.

Entender e configurar bem as propriedades é um passo fundamental para garantir que o CRM seja uma ferramenta eficaz no gerenciamento do relacionamento com o cliente e na organização de dados da sua empresa.
